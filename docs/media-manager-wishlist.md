# Media Manager Wishlist

Feature requests and UX priorities from Narin, the domain expert and co-founder. These reflect the needs of a media manager who works with evidence-based biohacking/health creators — someone who needs to produce, compare, and deliver audits to clients without touching technical configs.

---

## Must Have (Launch)

### 1. One-Click Audit Generation
Enter a handle, get a report. No config, no token budgeting, no model selection. The system figures out the rest. Narin should never see a terminal or know what a "prompt" is. The three-question flow (handle → goal → optional context) is the maximum acceptable input.

### 2. Peer Auto-Suggest
The system picks 3 same-tier comparable accounts automatically. Narin should never have to manually search for and input competitor handles — that's the engine's job. The auto-suggest includes rationale blurbs so Narin can explain to clients *why* these are their peers.

### 3. Beautiful Downloadable HTML
Reports that look professional enough to send directly to clients. No markdown artifacts, no raw data tables without styling. The report is the product — it must look like it came from a research institution, not a chatbot. The current Hemal audit design system is the quality bar.

### 4. Client-Ready Report Structure
The six-output framework (Where you're at / What's holding you back / Who's doing it better / What to post next week / When you hit the next milestone / The money move) should be the default. Narin shouldn't need to specify sections — the engine produces the right ones based on the goal question.

---

## Should Have (Month 1-2)

### 5. Report Comparison View
Side-by-side view of two audits for the same client (before/after implementing recommendations). This is Narin's primary client retention tool: "Here's where you were, here's where you are."

### 6. Batch Audit Export
Generate audits for multiple handles in one go (e.g., "audit these 5 accounts and give me all reports"). Useful for competitive landscape mapping and agency-style deliveries.

### 7. Custom Branding on Reports
Add client logo and "Prepared by [Narin / Agency Name]" footer to reports. The report should look like it came from Narin's consultancy, not from AuditLayer. White-label light.

### 8. Content Format Priority Override
Per-client toggle to adjust the content format priority: some biohacking creators use Reels more than Podcasts. The default ranking (Podcasts > Reels > Paper breakdowns) should be adjustable per audit.

---

## Nice to Have (Month 3+)

### 9. Mescreen Audit (Separate Product)
Instagram audit optimized for the Mescreen handle specifically. Different audience (consumers buying test kits vs. creators building brands), different goals (conversion vs. growth), different report structure. This is a separate product line, not a config toggle.

### 10. Client Portal
A dashboard where Narin's clients can log in to view their reports, track progress over time, and see comparison charts. Reduces Narin's delivery burden — clients self-serve their report history.

### 11. DM Script Generator
Given a creator's audit results, generate a personalized DM script Narin can use to reach out. "Hi [name], I ran an analysis of your Instagram presence and noticed [specific insight]. Would you be open to seeing the full report?" The script should pull specific data points from the audit to make it feel personal, not templated.

### 12. Report Annotation
Narin can add her own commentary/notes to auto-generated reports before sending to clients. The AI does the heavy lifting; Narin adds the human touch — "I'd add a Reel about [topic] because your audience specifically asked about this in DMs."

---

## Explicitly NOT Building

- **A social media management dashboard** — AuditLayer audits, it doesn't schedule posts. There are 50 scheduling tools. Not our lane.
- **Real-time analytics** — static snapshots, not live dashboards. The value is strategic depth, not minute-by-minute metrics.
- **AI chatbot for creators** — chat is for report refinement only, scoped to sections. No open-ended AI conversations.
- **Multi-platform content publishing** — not competing with Buffer/Hootsuite/Later.
