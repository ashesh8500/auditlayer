ALM

Design Rehaul & UX Calibration Specification

Author: Google Gemini 3.5 Agent Date: June 2026 Status: PASSED BUILD
Abstract: This document outlines the strategic UX/UI rehaul of AuditLayerMedia (ALM). Guided by design first principles, we identified critical friction gates in the client-onboarding funnel and cognitive overloads in our core report deliveries. In a new git worktree, we implemented and successfully compiled an interactive landing widget and a clean, tabbed report preview alongside a premium multi-step intake wizard.

1. Design Gaps & First Principle Reductions

The previous iteration of AuditLayerMedia featured solid aesthetic choices but suffered from structural flow gaps that impacted conversion and usability. Below is the mapping of identified gaps and their first-principles resolutions:

Aesthetic / Funnel Gap First Principle Applied Technical Resolution In Rehaul
High-Friction Landing Page: Clicking "Run Free Pulse Audit" hit a hard auth-wall at /login before users could input any details. Endowed Progress Effect / Goal Gradient Effect (Users are more committed once they initiate work). Created a live InteractivePulseWidget directly on the landing page. Users input handle, select goals, view a real-time scraping simulation, and see immediate scorecard insights before registering.
Static Scroll-heavy Mock Report: The `@glowstate` report was a giant 520px scrolling page, causing reader fatigue. Progressive Disclosure / Hick's Law (Reduce cognitive overload by hiding secondary parameters until needed). Designed a sleek InteractiveReportViewer with layout tabs (📊 Scores, 🎯 Gaps, 💡 Ideas, 📈 Plan) letting readers examine insights contextually.
Dry Step-by-Step App Wizard: The /audits/new wizard was basic and had no visual stepper or state transitions. Aesthetic-Usability Effect (Users perceive beautiful, responsive designs as more reliable). Re-engineered the wizard with an architectural stepper, custom platform icon integrations, live matching checks, and state glows.

2. Visual Tokens (Google DESIGN.md Spec)

We generated a complete Google-labs compliant DESIGN.md specification in the project root to formalize tokens. Contrast validation on npx @google/design.md lint DESIGN.md was performed to ensure WCAG AA compliance (4.5:1 ratio) on all interactive states, optimizing our primary CTA button background to #0f766e on `#ffffff` text.

3. Code Architecture & Component Mapping

The rehaul introduces pure component boundaries, shifting heavy clientside state into isolated modules while keeping App Router server benefits intact:

  • web/src/components/interactive-landing.tsx: Encapsulates all landing page interactivity including the simulated scanner, instant scoreboard, and mock report layout tabs.
  • web/src/app/page.tsx: Serves as the landing entrypoint checking user sessions and rendering components in a pristine semantic grid. Reduced from 690 lines of static markup down to 368 lines of highly maintained code.
  • web/src/app/(app)/audits/new/wizard.tsx: Manages the client-side intake stepper and connects seamlessly to the createAudit server action E2E.
